Spec sheet

Heavy machinery for lightweight design ops

Config, not code

One YAML file runs the factory

Every workflow lives in templates.yaml. Map Airtable field names to Figma layer names and the robot handles the rest. Adding a template is a config change, not a deploy of new code.

Multi-output

One trigger, a full asset kit

A single checked box can render the OG image, the LinkedIn card, and the speaker social — each to its own attachment field. A multi-select in Airtable picks which outputs run.

Smart variants

The layout picks itself

Second speaker photo filled? Two-speaker frame. Is Sponsored checked? Sponsored frame. Location says NYC? NYC frame. Variants compose — sponsored_two_speakers just works.

True to design

Your fonts, your pixels

Position, size, weight, color, and alignment are read from the Figma file itself and re-rendered with your actual TTFs. What the designer shipped is what the record gets.

Photo compositing

Headshots, handled

Attachments are pulled from the record, backgrounds removed with AI when you ask (remove_background: true), and clipped to the exact corner radius in Figma — circles included.

PDF reports

Multi-page documents too

Assemble static pages, data-overlay pages, and conditionally included pages — chosen by which field groups have data — into one branded PDF. Built for sponsor recap reports.

Operating manual

Feed it YAML, not tickets

The whole contract between your base and your design file fits on one screen: which table to watch, which frame to render, and how fields map to layers.

The poller claims records where the trigger is set, walks the status through Pending → Working → Done inside Airtable, and never lets one bad record stop the line.

What happens when you check the box

Airtabletrigger field set
▶
Clankerpoll · resolve variant
▶
Figmaexport frame · read layers
▶
Renderertext · photos · radius clip
▶
Airtableupload · status → Done

Figma's API can't edit files — so the frame is exported once as a base image, the layer geometry is read from the file JSON, and your data is redrawn on top, pixel-matched. Frame assets are cached, so a batch of fifty records hits Figma once.

Installation

Runs anywhere Python runs

Local

On your machine

Two dotfiles and a command. --run-once mode for testing a template.

uv sync && uv run airtable-to-figma
Container

Docker / Fly.io

Slim image with the background-removal model pre-baked. Ships with a ready fly.toml.

fly launch && fly deploy
Serverless

GitHub Actions cron

No server at all — a scheduled workflow runs one poll pass every ten minutes.

uv run airtable-to-figma --run-once
